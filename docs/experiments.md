# 实验记录与结果

为保持可重复性，请在此记录每次关键实验的配置、指标与备注。以下结构可直接复制扩展。

## 1. 记录模版

| 实验编号 | 配置文件 | 关键改动 | 数据集 | 训练轮数 | 采样器 | PSNR (dB) | SSIM | 备注 |
| -------- | -------- | -------- | ------ | -------- | ------ | --------- | ---- | ---- |
| EXP-000 | `configs/sr_train.json` | 基线 | `train.h5` | 1000 | DDIM-20 | - | - | 首次基线，待填结果 |

填写建议：

- **实验编号**：递增或按日期命名，例如 `EXP-20241028-A`。
- **配置文件**：对应 `configs/` 下的 JSON 名称（请提交到版本库）。
- **关键改动**：仅列出区别于基线的核心变化（例如更换骨干、调节 `ddim_steps`、启用数据增强）。
- **数据集**：注明数据版本，如 `train_v1.h5`、`train_augmented.h5`。
- **训练轮数**：实际完成的 epoch 数或 early stop 时刻。
- **采样器**：`DDIM-20 (eta=1.0)`、`DDPM-1000` 等。
- **指标**：PSNR/SSIM 取自 `results.txt` 或 `results_per_image.csv` 的平均值；可附加标准差。
- **备注**：记录异常、硬件信息、阈值设置、训练耗时等补充信息。

## 2. 示例参考

- **骨干对比**：分别训练 `convnet_small` / `convnet_big` / `unet_res`，比较收敛速度与最终指标。
- **采样步数**：在评估中尝试 `ddim_steps = 10/20/50`，观察速度与质量权衡。
- **数据增强**：对比 `augment=false/true`、`use_tfm_channels=true/false` 的差异。

请在完成评估后及时更新以上表格，并将 `SR/eval_results_xxx` 中的 `config.json`、指标 CSV、示例图像（如有）与实验编号对应，方便团队共享。

若实验引入新模块（例如自定义骨干或 VAE），建议在《FAQ》同步新增常见问题与排查经验。
